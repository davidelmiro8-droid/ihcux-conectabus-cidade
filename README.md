Descrição do Projeto

O ConectaBus é um aplicativo de mobilidade urbana desenvolvido para auxiliar usuários no deslocamento diário utilizando transporte público.

O aplicativo reúne informações importantes em um único lugar, permitindo:

buscar rotas;
acompanhar trajetos;
visualizar tempo de chegada;
consultar lotação dos veículos;
utilizar carteira digital com QR Code;
receber alertas;
e acessar recursos de segurança.

O principal objetivo é tornar a experiência de mobilidade mais rápida, segura e organizada.

Contexto de Uso

O aplicativo foi pensado para usuários que utilizam transporte público em situações de pressa e movimento constante.

O usuário pode estar:

correndo para não perder o ônibus;
segurando mochila ou sacolas;
utilizando o celular com apenas uma mão;
em locais movimentados e barulhentos;
ou precisando tomar decisões rápidas durante o trajeto.

Por isso, o design prioriza simplicidade, clareza e rapidez de navegação.

Decisões de UX

O protótipo foi desenvolvido com foco em acessibilidade, rapidez e prevenção de erros de navegação.

As informações mais importantes, como:

tempo de chegada;
rota selecionada;
lotação do veículo;
e QR Code de pagamento;

receberam maior destaque visual para facilitar a leitura rápida.

Além disso:

os botões possuem tamanho maior para facilitar o toque;
o fluxo possui poucas etapas;
as telas apresentam organização simples;
e as ações principais permanecem sempre visíveis.

O layout também foi pensado para uso com apenas um polegar, considerando usuários em movimento ou segurando objetos.

Acessibilidade

O aplicativo foi projetado para facilitar o uso por pessoas com mobilidade reduzida ou em situações de pressa.

Para isso:

os elementos clicáveis possuem tamanho maior;
os botões ficam posicionados em áreas de fácil alcance;
as informações essenciais possuem destaque visual;
e os fluxos foram simplificados para reduzir erros.

A tela de detalhes do veículo também apresenta informações sobre acessibilidade, como presença de rampas e recursos disponíveis no transporte.

Além disso, a área de alertas possui botão de emergência para situações de risco.

Telas do Protótipo
1. Busca de Rota

Tela inicial utilizada para pesquisar destinos e acessar locais frequentes.

2. Resultado de Rotas

Apresenta diferentes opções de trajetos com comparação de tempo, preço e baldeações.

3. Mapa de Percurso

Mostra o trajeto em tempo real e o deslocamento do usuário durante a viagem.

4. Carteira Digital

Exibe saldo disponível, QR Code para pagamento e histórico de transações.

5. Detalhes do Veículo

Apresenta informações sobre lotação, acessibilidade, segurança e tempo de chegada.

6. Alertas

Tela responsável por exibir avisos importantes, interrupções e botão de ajuda.

Fluxo Principal
O usuário acessa a tela de busca de rota.
Digita o destino desejado.
O aplicativo apresenta diferentes opções de trajetos.
O usuário escolhe a melhor rota.
O mapa mostra o percurso em tempo real.
O usuário acompanha informações do veículo.
Na carteira digital, utiliza o QR Code para validar a passagem.
Reflexão de IHC

Em sistemas de mobilidade urbana, um erro pode fazer o usuário perder o transporte ou seguir para o lado errado da cidade.

Por isso, o protótipo foi desenvolvido com foco em:

clareza visual;
confirmação das ações;
navegação simples;
e destaque para informações importantes.

As ações principais permanecem sempre visíveis e o sistema fornece feedback constante ao usuário, reduzindo erros e aumentando a confiança durante o uso.

Além disso, o layout foi planejado para permitir uso rápido com apenas uma mão, mesmo em ambientes movimentados.
